\begin{DoxyWarning}{Warning}
It can be O\+N\+L\+Y 0 or 1! O\+V\+E\+R\+W\+R\+I\+T\+E==1 is required when \char`\"{}string\char`\"{} functions (\doxyref{bt\+\_\+get\+Str}{p.}{bluetooth_8h_ad9503dbabb1f203a372431cb89c3d1f0}, \doxyref{bt\+\_\+send\+Str}{p.}{bluetooth_8h_ad0c7b930ecc1ce44a99e41a0a7cc7fff}) are used. Otherwise Tx buffer lock may occur (\doxyref{bt\+\_\+get\+Str}{p.}{bluetooth_8h_ad9503dbabb1f203a372431cb89c3d1f0} uses \doxyref{string\+\_\+count}{p.}{bluetooth_8h_ae0b07eb3386d833f10670a32c37926e8} which is incremented by N\+U\+L\+L incoming character. Functions guarantee the continuity of strings, but use it carefully. Best way is to use still same type of functions (for example \doxyref{bt\+\_\+get\+Char}{p.}{bluetooth_8h_aaf6401bea615560b2317eadeec074670}, \doxyref{bt\+\_\+send\+Char}{p.}{bluetooth_8h_a6231f952dfce20a3235815eb0143cced} / \doxyref{bt\+\_\+get\+Str}{p.}{bluetooth_8h_ad9503dbabb1f203a372431cb89c3d1f0}, \doxyref{bt\+\_\+send\+Str}{p.}{bluetooth_8h_ad0c7b930ecc1ce44a99e41a0a7cc7fff}).
